# רכזת הקישורים של איתן

אתר סטטי קטן שכל תפקידו לרכז את הקישורים שלך במקום אחד יפה — עם ממשק ניהול פרטי
שמאפשר להוסיף ולערוך קישורים **בלי לגעת בקוד**. בנוי לרוץ על **GitHub Pages** בלבד,
בלי שרת ובלי מסד נתונים חיצוני.

- **עמוד ציבורי:** `https://<user>.github.io/links/`
- **ניהול (רק אתה):** `https://<user>.github.io/links/admin.html`

איך זה עובד: הקישורים נשמרים בקובץ `data/links.json` בתוך ה-repo. עמוד הניהול כותב
לקובץ הזה ישירות דרך ה-GitHub API (commit), ו-GitHub Pages בונה מחדש את האתר. ה"התחברות"
היא Personal Access Token שלך — רק לבעל ה-repo יש כזה, אז רק אתה יכול לשמור.

---

## 1) העלאה ל-GitHub Pages

1. העתק את **כל התיקייה הזו** לתוך ה-repo `‏<user>.github.io` בשם התיקייה **`links`**
   (כלומר הקבצים יגיעו ל-`links/index.html`, `links/admin.html`, וכו').
2. בצע commit ו-push לענף `main`:
   ```bash
   git add links
   git commit -m "Add link hub"
   git push
   ```
3. אחרי דקה, האתר יהיה זמין ב-`https://<user>.github.io/links/`.

> רוצה שם אחר מ-`links` (למשל `me` או `hub`)? פשוט תן לתיקייה שם אחר — הקוד מזהה את
> הנתיב אוטומטית, אין מה לשנות.

---

## 2) יצירת טוקן (פעם אחת)

1. היכנס ל-**GitHub → Settings → Developer settings → Fine-grained tokens →
   Generate new token** (או ישירות: <https://github.com/settings/personal-access-tokens/new>).
2. **Token name:** משהו כמו `link-hub`.
3. **Expiration:** לבחירתך (למשל 90 יום; אפשר לחדש בהמשך).
4. **Repository access → Only select repositories →** בחר את `‏<user>.github.io` בלבד.
5. **Permissions → Repository permissions → Contents → Read and write**.
6. Generate token → **העתק** את הטוקן (הוא מוצג פעם אחת).

עכשיו פתח את `‏.../links/admin.html`, הדבק את הטוקן, לחץ **כניסה** — וזהו. הטוקן נשמר רק
בדפדפן הזה. בכל כניסה הבאה מאותו דפדפן לא תצטרך להזין אותו שוב.

---

## 3) שימוש יומיומי

- **הוספת קישור:** בעורך, לחץ **+ הוסף קישור**, מלא שם + כתובת (ואם בא לך תיאור/אמוג׳י).
- **סדר:** גרור בעזרת הנקודות, או השתמש בחצים מעלה/מטה.
- **הסתרה זמנית:** מתג "מוסתר" — הקישור נשמר אבל לא מוצג בעמוד הציבורי.
- **קבוצות:** אפשר לקבץ קישורים תחת כותרת (לא חובה — בלי כותרת זו פשוט רשימה).
- **שמירה:** כפתור **שמירה** (או Ctrl/Cmd+S). השמירה יוצרת commit ל-repo.

> ⏱️ אחרי שמירה יש עיכוב של ~10–60 שניות עד שהעמוד הציבורי מתעדכן — זה זמן הבנייה של
> GitHub Pages. בעורך רואים את השינוי מיד.

**כפתור נעילה** מוחק את הטוקן מהדפדפן (שימושי במחשב משותף).

---

## 4) התקנה כאפליקציה (PWA)

- **אתר ציבורי:** פתח את `‏.../links/` בכרום/edge → בסרגל הכתובת יופיע אייקון התקנה →
  "התקן". במובייל: תפריט → "הוספה למסך הבית".
- **ניהול:** אותו דבר מתוך `‏.../links/admin.html` — זו מתקינה כאפליקציה נפרדת (אייקון
  ושם שונים) כך שהניהול יושב אצלך כאפליקציה בפני עצמה.

---

## 5) התאמות אישיות

- **החלפת התמונה/אווטאר:** שנה את הכתובת בשדה "כתובת תמונת פרופיל" בעורך, או החלף את
  הקובץ `assets/icons/avatar.svg`.
- **צבעים:** כל הצבעים מרוכזים בראש `assets/css/styles.css` (חפש `--accent`, `--bg` וכו').
- **פונט:** הפונט (Assistant) מוטמע ב-`assets/fonts/`. אפשר להחליף בקבצי woff2 אחרים.
- **אייקוני אפליקציה:** ב-`assets/icons/` (`icon-192/512`, `apple-touch-icon`,
  ומקבילי ה-`admin-`). לחידוש: יש סקריפט לדוגמה שנוצר בזמן הבנייה.

---

## 6) פיתוח מקומי

מכיוון שהאתר קורא קבצים, צריך להריץ שרת סטטי (לא לפתוח את הקובץ ישירות):

```bash
npx serve .
# או:
python -m http.server 8000
```

ואז לפתוח `http://localhost:8000/`. במצב מקומי (לא github.io) **העורך פועל לתצוגה
מקדימה בלבד** — אי אפשר לשמור, כי שמירה מחייבת את ה-API של GitHub. זה מכוון.

---

## אבטחה בקצרה

- הטוקן נשמר רק ב-`localStorage` של הדפדפן שלך ונשלח אך ורק ל-`api.github.com`.
- הוא לא מוטמע בקוד ולא מופיע באתר — מבקר בעמוד הניהול ללא טוקן פשוט לא יכול לשמור.
- עמוד הניהול מסומן `noindex` ואינו מקושר מהעמוד הציבורי; בעמוד הציבורי אין כפתור התחברות.
- מומלץ טוקן עם תוקף (expiry) ומוגבל ל-repo הזה בלבד עם הרשאת Contents בלבד.
